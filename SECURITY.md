# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in `dexter`, please **do not** open a public issue. Instead, report it privately by:

1. **Email**: yuki.takami@tec-co.jp
2. **GitHub Private Vulnerability Reporting**: https://github.com/YukiTakamitec/dexter/security/advisories/new

We aim to acknowledge reports within **72 hours** and to provide an initial assessment within **7 days**.

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest `main` | :white_check_mark: |
| older releases | :x: |

## Disclosure Policy

- We follow a **coordinated disclosure** process.
- Once a fix is ready, we will publish a security advisory through GitHub Security Advisories.
- Credit will be given to reporters who wish to be acknowledged.

## Security Practices

- All dependencies are monitored via Dependabot alerts.
- Secret scanning is enabled on this repository.
- Default branch is protected; changes require pull request review.
